## Syfte

Vecka 5 handlar om projektarbete där fokus ligger på att vidareutveckla och förbättra CV-sidan som helhet.

Målet är att arbeta mer sammanhängande med projektet genom att:
- förbättra struktur och innehåll
- tänka på tillgänglighet och enkel användarupplevelse (UX)
- förbereda webbplatsen för publicering

Denna vecka handlar mindre om nya tekniker och mer om att använda det ni redan kan på ett genomtänkt och professionellt sätt.
 
---

## Hur ska vi tänka denna vecka?
- Arbeta med det projekt som redan finns
- Förbättra istället för att bygga nytt
- Göra små justeringar som höjer helhetsintrycket
- Tänka på tillgänglighet och användarvänlighet
- Se till att sidan känns färdig och redo att visas upp

---

## Vad gör vi under veckan?
- Går igenom hela webbplatsen och ser till att allt fungerar
- Förbättrar tillgänglighet (rubriker, alt-text, kontrast, fokus)
- Gör enkla UX-förbättringar
- Rensar och strukturerar kod vid behov
- Förbereder projektet för publicering

## Pseudokod – hur arbetet med Git är uppbyggt (Vecka 5)

ARBETSCYKEL (när vi gör förbättringar i projektet)
  1) git pull
     - hämta senaste versionen från GitHub

  2) Gör en förbättring (en sak i taget)
     - t.ex. tillgänglighet (alt, rubriker, fokus, kontrast)
     - t.ex. UX (aktiv länk i meny)

  3) Testa i webbläsaren
     - fungerar allt som det ska?

  4) git status
     - se vilka filer som ändrats

  5) git add .
     - lägg till ändringar

  6) git commit -m "kort och tydligt"
     - spara en version (en förbättring per commit)

  7) git push
     - ladda upp till GitHub

EXEMPEL PÅ COMMITS DENNA VECKA
  - "Add h1 to home page"
  - "Add alt text to hero image"
  - "Fix heading order in hero"
  - "Add focus-visible styles"
  - "Highlight active menu link"
  - "Improve muted text contrast"
